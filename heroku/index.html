📌 Обзор проекта Heroku

Heroku — это продвинутый Telegram userbot с улучшенной безопасностью и современными функциями. Основан на Hikka, но имеет дополнительные улучшения и исправления.

💡 Важно: Этот проект является форком Hikka от coddrago с дополнительными патчами и улучшениями.

⚙️ Установка

VPS/VDS (Ubuntu/Debian)

# Обновление и установка зависимостей sudo apt update && sudo apt install git python3 -y # Клонирование репозитория git clone https://github.com/coddrago/Heroku cd Heroku # Создание виртуального окружения python3 -m venv .venv source .venv/bin/activate # Установка зависимостей pip install -r requirements.txt # Запуск python3 -m heroku

Аргументы запуска

  • --proxy-pass — открыть SSH туннель для веб-интерфейса
  • --no-web — установить без веб-интерфейса (консоль)
  • --root — для пользователей root

📋 Требования

  • Python: 3.10+
  • API: API_ID и API_HASH с my.telegram.org
  • ОС: Linux, WSL, Android (UserLAnd)

📊 pointer() - Указатели на данные

📂 Исходник: heroku/types.py → Module.pointer()

Создает "умный" указа��ель на данные в БД. Изменения автоматически сохраняются.

module.py
@loader.tds
@loader.owner
class UsersModule(loader.Module):
    strings = {"name": "UsersModule"}

    def __init__(self):
        # Указатель на список
        self.users = self.pointer("registered_users", [])

    async def register_user(self, user_id: int, name: str):
        # Добавление - автоматически сохраняется в БД
        self.users.append({
            "id": user_id,
            "name": name
        })
        # Не нужно вызывать db.set()!

    async def unregister_user(self, user_id: int):
        # Удаление
        self.users = [u for u in self.users if u["id"] != user_id]

    async def get_all_users(self):
        return self.users
